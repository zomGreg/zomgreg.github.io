Cloud Management Value Propositions

 Mon 20 January 2014 by Greg Moselle

I was recently challenged to distill the value propositions of a cloud management technology into three success criteria. Consumers of cloud management technology are interested in:

1. Multi-cloud Management. The ability to manage multiple cloud accounts with a single user interface and API.

2. Bring self-provisioning to the end user. Empower the demand-side of cloud resources to provision their own resources quickly and securely.

3. Transparency for managers. Enable managers to gain programmatic insight into cloud use.

Multi-Cloud Management

Multi-cloud management is no small technological success. Bringing the ability to manage disparate cloud technologies under a single user interface is orders of magnitude more empowering that managing several email accounts with a single application. While the user interface typically garners the most attention, I'd argue the API is the most interesting facet of the multi-cloud management platform.

Consider the agility a mutli-cloud API brings to an organization. One customer I'm currently working with is interested in being able to "fail fast". Take that objective for what it's worth, but it's sound engineering practice. A multi-cloud API allows for rapid cross-cloud prototyping without all the overhead of building separate modules for each cloud. The API is where it's possible bring the best of what each cloud has to offer to the end user.

Self Provisioning

Bringing self-provisioning to the end user is where there is the most ground to be gained in winning cloud technology converts. For example, I've experienced lead times for procuring compute resources that extend up to 4 weeks for a virtual server.

I've also heard anecdotal reports of up to 6 weeks for the same. Empowering users to self-provision compute resources such as virtual machines and storage on the order of minutes will leave them feeling like they've seen the face of god.

Transparency

Finally, with all the self-provisioning that accompanies user enablement, a need for transparency will arise. The ability to gain insight into cloud use/adoption is valuable for understanding the costs involved and how consumers are actually using the technology.

While cost might be the most obvious metric for tracking cloud infrastructure use, another metric that may be equally compelling is virtual machine lifetime. Traditionally, infrastructure is provisioned and left to exist for as much time as possible, even if the original reason for the procurement has changed. Furthermore, the longer the length of time for which a piece of infrastructure exists is often a viewed as a positive indicator.

In a cloudy infrastructure environment, the opposite metric can apply. One manager of a virtual infrastructure environment told me the metric she's tracking for VM lifetime is 30 days. Any machine in existence for longer than 30 days is viewed as a candidate for termination because it's more likely that VM is forgotten and lost in the cloud than performing a useful function.

Multi-cloud, self-provisioning, and visibility are three key value propositions for a successful cloud management solution.
